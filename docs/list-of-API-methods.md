# API Endpoints — Сервис предзаказа и самовывоза еды MAVIN (MVP)

## Общие принципы

- **Базовый URL:** `/api/v1`
- **Формат:** JSON
- **Авторизация:** Bearer Token (JWT) в заголовке `Authorization`
- **Модель работы:** Предзаказ и самовывоз (Click & Collect). Модуль курьеров отсутствует.
- **Хранилище файлов:** S3-совместимое (MinIO). Загрузка через `POST /upload`, возвращает URL объекта.
- **Пагинация:** Все GET-эндпоинты, возвращающие списки, поддерживают query-параметры `page` (default 1) и `limit` (default 20).
- **Уведомления:** WebSocket-соединение `/ws/orders` — клиент получает события смены статуса заказа в реальном времени.
- **Роли доступа:** обозначены иконками:
  - 🌐 — публичный (без авторизации)
  - 👤 — покупатель (customer)
  - 🍽️ — администратор ресторана (restaurant_admin)
  - 🔑 — администратор сайта (site_admin)

## 8. Заказы

### Логика скидок при оформлении

При `POST /orders` бэкенд:
1. Если передан `promo_code` — валидирует его (проверяет существование, срок, `is_active`, не использован ли этим пользователем). Если невалиден — ошибка 404.
2. Если пользователь `is_premium` — доступна скидка подписки 5%.
3. **Скидки НЕ складываются.** Применяется наибольшая из двух: промокод или скидка подписки.
4. Итого: `total = subtotal - (subtotal * max(promo_discount, subscription_discount) / 100)`.

### Для клиента

| Метод | Endpoint | Доступ | Описание |
|-------|----------|--------|----------|
| POST | `/orders` | 👤 | Оформить заказ. Тело: `pickup_time` (время самовывоза), `comment`, `payment_method` (`card_online` / `cash_on_receipt`), `promo_code` (опционально). Если блюдо из корзины закончилось — ошибка 409. Если промокод невалиден — ошибка 404. Корзина очищается |
| GET | `/orders` | 👤 | История заказов. Query: `page`, `limit`. Поля: `id`, `date`, `total_price`, `restaurant_name`, `status` |
| GET | `/orders/{id}` | 👤 | Детали заказа (состав, статус, адрес самовывоза, скидка) |
| POST | `/orders/{id}/cancel` | 👤 | Отменить заказ. Работает только в статусе `created`. Иначе — ошибка 422 |

### Для ресторана (Обработка)

При смене статуса заказа бэкенд отправляет WebSocket-событие клиенту.

| Метод | Endpoint | Доступ | Описание |
|-------|----------|--------|----------|
| GET | `/owner/orders` | 🍽️ | Список активных заказов ресторана (определяется по JWT администратора ресторана). Заказы premium-клиентов — первыми. Query: `status`, `page`, `limit` |
| GET | `/owner/orders/{id}` | 🍽️ | Детали заказа |
| POST | `/owner/orders/{id}/accept` | 🍽️ | Принять в работу → статус `cooking`. WS-событие клиенту |
| POST | `/owner/orders/{id}/ready` | 🍽️ | Собрано → статус `ready_for_pickup`. WS-событие: "Можно забирать" |
| POST | `/owner/orders/{id}/complete`| 🍽️ | Выдано клиенту → статус `completed`. WS-событие клиенту |

## 10. AI-Ассистент (Нутрициолог)

Бэкенд использует context injection: получает меню текущего ресторана из БД (с учётом аллергенов пользователя из профиля), формирует system prompt с контекстом, отправляет в LLM. История запросов сохраняется в БД для внутренней аналитики (не доступна через API).

| Метод | Endpoint | Доступ | Описание |
|-------|----------|--------|----------|
| POST | `/ai/recommend` | 👤 (premium) | Запрос к LLM. Тело: `prompt`, `restaurant_id`. Возвращает ответ текстом и массив `recommended_dish_ids`. Ошибка 403, если нет подписки. Ошибка 503, если LLM-сервис недоступен |

## 15. WebSocket — Уведомления о статусе заказа

| Тип | Endpoint | Доступ | Описание |
|-----|----------|--------|----------|
| WS | `/ws/orders` | 👤🍽️ | Подключение по WebSocket с JWT-токеном (query-параметр `token`). Сервер отправляет JSON-события при смене статуса заказа |

### Формат события (сервер → клиент)

```json
{
  "event": "order_status_changed",
  "data": {
    "order_id": 4501,
    "new_status": "ready_for_pickup",
    "message": "Ваш заказ готов! Можно забирать."
  }
}
```

### Кто получает:
- **Покупатель** — события по своим заказам (`cooking`, `ready_for_pickup`, `completed`)
- **Администратор ресторана** — события по новым заказам (`created`) в своём ресторане

## Коды ошибок

| Код | Когда возникает |
|-----|-----------------|
| 400 | Невалидные данные (пустые поля, неверный формат времени самовывоза, файл > 5 МБ) |
| 401 | Не авторизован (нет токена или он истёк) |
| 403 | Нет доступа (попытка вызвать `/ai/recommend` без подписки; заблокированный пользователь) |
| 404 | Ресурс не найден (блюдо удалено, ресторан скрыт, промокод не существует или истёк) |
| 409 | Конфликт (блюдо в корзине из другого ресторана; блюдо закончилось при оформлении заказа) |
| 422 | Логическая ошибка (попытка отменить заказ, который уже готовится) |
| 500 | Внутренняя ошибка сервера |
| 503 | Внешний сервис недоступен (LLM API не отвечает) |

## Жизненный цикл заказа (State Machine)

```text
created (создан) → cooking (готовится) → ready_for_pickup (ждет выдачи) → completed (выдан)
   ↓
cancelled (отменен клиентом)
```

При каждом переходе статуса — WebSocket-событие заинтересованным сторонам.
